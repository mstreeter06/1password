# Vault User & Group Access Report (updated)

A small utility to generate CSV reports showing which users and groups have access to your 1Password vaults. This was developed for our csv file that was then uploaded into our SailPoint Identity Security Cloud tenant for our User Access Reviews. This is an updated version of 1Password's official example; see the credit section for the original source.

**Prerequisites:**
- Python 3.8+ (run with `python3`)
- 1Password CLI (`op`) installed and signed in. Install and get started: https://developer.1password.com/docs/cli/get-started/

**Script:**
- `vault-user-group-access-report-updated.py` — main script (located in the same directory as this README).

**Basic usage:**

Run the script from the directory containing the script:

```
python3 vault-user-group-access-report-updated.py [options]
```

Important command-line options:
- `--no-backup` : Disable backing up existing per-vault CSV files.
- `--backup-retention N` : Keep the last N backup runs (default set in script).
- `--fresh-run` : Remove existing vault CSV files and re-process all vaults.
- `--vault-filter "<regex>"` : Process only vaults whose name matches the regex.
- `--user-name-filter "<regex>"` : Filter final combined report by user name.
- `--user-email-filter "<regex>"` : Filter final combined report by user email.
- `--case-sensitive` : Make filters case-sensitive (default is case-insensitive).

See in-script `parse_args()` for up-to-date option names and behaviour.

**Outputs / layout:**
- `vault_reports/` — per-vault CSV files and an `exports/` subfolder with timestamped copies.
- `vault_access_report.csv` — combined report generated by the script (or `vault_access_report_filtered.csv` when filters applied).
- `vault_reports/backups/` — timestamped backups of previous per-vault CSVs (if backups enabled).
- `logs/` — `run-*.txt`, `issues-*.txt`, `issues-*.csv`, `stats-*.txt` (script run logs and issues).
- Progress checkpoint files: `vault_reports/progress_<timestamp>.json` allow resuming on restart.

**Behavior notes:**
- The script uses the `op` CLI and will exit if `op` is missing or not authenticated. If your 1Password session expires while the script is running, it will raise an authentication error and save progress so you can sign in and restart.
- By default the script skips vaults that already have per-vault CSV files; use `--fresh-run` to force re-processing.

**Examples:**
- Basic full run (writes per-vault CSVs, combined report, and logs):

```bash
python3 vault-user-group-access-report-updated.py
```

- Fresh run (remove existing per-vault CSVs and re-process all vaults):

```bash
python3 vault-user-group-access-report-updated.py --fresh-run
```

- Run with a vault name filter (regex) and produce a filtered final report by user email:

```bash
python3 vault-user-group-access-report-updated.py --vault-filter "^Engineering" --user-email-filter "@example.com$"
```

**Sample combined report (CSV) header and sample rows:**

```
vaultName,vaultUUID,name,email,userUUID,status,assignment,permissions
Engineering Vault,abcd-1234,Jane Doe,jane.doe@example.com,user-1111,ACTIVE,Direct,read,write
Engineering Vault,abcd-1234,John Smith,john.smith@example.com,user-2222,ACTIVE,Group (SRE),read
```

**Notes:**
- The script itself has no external Python package dependencies — it uses the standard library. The only external requirement is the 1Password CLI (`op`).
